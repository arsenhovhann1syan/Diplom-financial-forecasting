# ԴԻՊԼՈՄԱՅԻՆ ԱՇԽԱՏԱՆՔ
**ԹԵՄԱ՝** Ֆինանսական շուկայի վերլուծություն արհեստական բանականության միջոցով  
**Ուսանող՝** Արսեն Հովհաննիսյան  
**Համալսարան՝** Հայաստանի ազգային պոլիտեխնիկական համալսարան  
**Աստիճան՝** Բակալավր  
**Բաժին՝** Կիրառական մաթեմատիկա  

---

## ԲՆՈՒԹԱԳԻՐ
Ֆինանսական շուկայի ճշգրիտ վերլուծությունը և կանխատեսումը կարևոր դեր ունեն ներդրումային որոշումներ կայացնելու և ռիսկերը նվազեցնելու գործում։ Այս աշխատանքը նպատակ ունի ստեղծել ամբողջական համակարգ Bitcoin-ի (և հնարավոր այլ Altcoin-ների) գնային փոփոխությունները կանխատեսելու համար՝ օգտագործելով Արհեստական Բանականության (AI) համալիր մեթոդներ։ Ուսումնասիրվում են թե՛ ավանդական ժամանակային շարքերի մոդելները (ARIMA, GARCH), թե՛ ժամանակակից Մեքենայական Ուսուցման (Random Forest, XGBoost) և Deep Learning (RNN, LSTM) մոտեցումները։  
Գործնական մասը բաժանված է մի շարք քայլերի, որոնք ընդգրկում են տվյալների հավաքումից մինչև մոդելների գնահատում և հուսալիության ապահովում՝ production-ին մոտ միջավայրում։

## ՆՊԱՏԱԿՆԵՐ ԵՎ ԽՆԴԻՐՆԵՐ

### Գլխավոր Նպատակ
Ստեղծել ամբողջական կանխատեսման համակարգ, որը կարող է կանխատեսել Bitcoin-ի գինը 1–24 ժամ հորիզոնում՝ ապահովելով մոդելների ստատիստիկորեն հիմնավորված համեմատություն և production-ին մոտ մշտադիտարկում։

### Նպատակներ
- Ֆինանսական տվյալների ավտոմատ հավաքում և պահպանման համակարգ ստեղծել (Data Collection).  
- Տվյալների որակի ապահովում՝ բացակա արժեքներ, սխալներ և անոմալիաներ հայտնաբերելով (Data Validation).  
- Ստեղծել նոր հատկանիշներ՝ տեխնիկական ինդիկատորներ, lag-եր, rolling statistics և ժամանակային հատկություններ (Feature Engineering).  
- Տվյալների վիզուալիզացիա և ուսումնասիրություն՝ correlations, distribution, trend, seasonality (EDA).  
- Թրեյնինգ և թեստավորում՝ տարբեր մոդելների համեմատության համար (Model Training & Evaluation).  
- AB Testing մոդելների արդյունավետության ճշտման համար, կարևորությունը չափելու միջոցով (T-test, bootstrap և այլն).  
- Ստեղծել ensemble մոդելներ՝ բազմամոդելային կանխատեսումներ բարելավելու համար (Ensemble Modeling).  
- Ստեղծել probabilistic forecasting մեթոդներ՝ confidence intervals հաշվարկելու համար.  

### MLOps & Production-ready բաղադրիչներ
- Մոդելի պահպանում և Model Registry մոդելի տարբեր տարբերակները պահվում են, որոնց կարելի է հետագայում հասնել և օգտագործել.  
- Մոդելի պրոֆիլինգ և վերահսկում (Production Monitoring) – իրական ժամանակում հետևում ենք մոդելի աշխատանքի վիճակին՝ հայտնաբերում փոփոխություններ, չափում կատարողականը (rolling metrics) և ստեղծում ահազանգեր (alerts) խնդիրների դեպքում.  
- Միավորում ենք բոլոր մոդուլները՝ պատրաստելով ամբողջական համակարգ՝ տվյալների պատրաստում → մոդելի թրեյնավորում → գնահատում → պահպանում → կանխատեսումներ → մոնիթորինգ, որպեսզի ստացվի պատրաստ օգտագործման և թեստավորման համար:

### Real-Time Signal (On-Demand Manual Trading)
Վերջնական քայլ՝ օգտագործողի համար:

## Տվյալների աղբյուրներ
- Պատմական շուկայային տվյալներ  
- Ֆինանսական Ակտիվներ: Bitcoin (BTC), որպես հիմնական թիրախ, և հնարավոր է այլ Altcoin-ներ (օրինակ՝ Ethereum կամ Solana)  
- Տվյալների Տեսակը: OHLCV (Open, High, Low, Close, Volume) տվյալներ՝ ցանկալի է բարձր հաճախականությամբ  
- API Ինտեգրում: Տվյալների հավաքագրումը կիրականացվի API ինտերֆեյսների միջոցով (օրինակ՝ Binance API)  
- MLOps Կապը: API-ի օգտագործումը թույլ է տալիս ճշգրտորեն իրականացնել MLOps Փայփլայնի առաջին քայլը (Քայլ 1 - Data Collection), քանի որ մոդելը մարզվելու է այն նույն ճանապարհով ստացված տվյալներով, ինչպես իրական արտադրական միջավայրում։
